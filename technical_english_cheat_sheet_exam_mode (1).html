IT Grammar & Technical Structures
▸ Modal Verbs in Technical Context

Obligation / Necessity

MUST / HAVE TO
The firewall must block all inbound traffic on port 23.
O firewall deve bloquear todo o tráfego de entrada na porta 23.
You have to apply the patch before Monday.
Você precisa aplicar o patch antes de segunda.
MUST = internal necessity / policy. HAVE TO = external requirement.

Prohibition

MUST NOT / CANNOT
Users must not share credentials under any circumstances.
Usuários não devem compartilhar credenciais em hipótese alguma.
You cannot access the production environment without MFA.

Recommendation / Best Practice

SHOULD / OUGHT TO
You should enable logging on all network devices.
Você deveria ativar o registro em todos os dispositivos de rede.
Passwords ought to be at least 12 characters long.

Possibility / Capability

CAN / COULD / MAY / MIGHT
The attacker may exploit this misconfiguration.
O atacante pode explorar essa configuração incorreta.
This vulnerability could lead to remote code execution.
▸ Passive Voice in IT Documentation

Technical docs heavily use passive voice. Learn to recognize and produce it.

Active → Passive Examples

Active: The administrator configured the ACL.
→ Passive: The ACL was configured by the administrator.
Active: The IDS detected an intrusion.
→ Passive: An intrusion was detected by the IDS.
Active: Engineers are patching the server.
→ Passive: The server is being patched.

Common Passive Structures in IT

The router has been configured with OSPF.
Traffic is filtered by the ACL.
The certificate was revoked after the breach.
Ports are scanned periodically by the security team.
All logs must be retained for 90 days.
▸ Conditional Sentences in Troubleshooting

Type 1 — Real Conditions (If present → will)

If the interface is down, the route will be removed from the routing table.
If you enable port security, the switch will block unknown MAC addresses.

Type 2 — Hypothetical (If past → would)

If the attacker had valid credentials, they would bypass the firewall entirely.
If we used HTTPS, the traffic wouldn't be readable in transit.

Mini Reading Strategy

1. Read the question first.
2. Identify key verbs and signal words.
3. Eliminate grammatically impossible options.
4. Choose the most natural sentence in context.
5. Re-read the full sentence before answering.
▸ Short Reading Examples

Operational English

The monitoring system detected unusual outbound traffic from a workstation. The analyst reviewed the logs, confirmed the anomaly, and isolated the host from the network.
Key verbs: detected, reviewed, confirmed, isolated.

Report English

The vulnerability was identified during routine testing. Immediate remediation is recommended to reduce the risk of exploitation and prevent lateral movement.
Notice the passive voice: was identified, is recommended.

▸ Certification Exam English — Tricky Phrases

Common Exam Qualifiers

"BEST describes"Look for the most accurate, complete answer.
"MOST likely"Choose the probable, not the possible.
"FIRST step"Process/order matters. Think methodically.
"LEAST privilege"Minimum access needed — common security principle.
"EXCEPT"All options are correct EXCEPT one. Read carefully!
"ALWAYS / NEVER"Absolute qualifiers — usually wrong in nuanced contexts.
"by default"Factory/default configuration, no changes made.
"in transit"Data moving across a network (vs "at rest").

Tricky Word Pairs

authentication vs authorizationAuth-N = who you are. Auth-Z = what you can do.
encryption vs encodingEncryption = secret key. Encoding = no key (e.g., Base64).
IDS vs IPSIDS detects (passive). IPS prevents (inline, active).
vulnerability vs exploitVuln = weakness. Exploit = attack leveraging it.
integrity vs non-repudiationIntegrity = data unchanged. Non-repudiation = can't deny action.
symmetric vs asymmetricSymmetric = same key. Asymmetric = key pair.
